# 산출물 형식

모든 산출물은 `outputs/` 아래에 만든다. CSV는 UTF-8로 작성하고, 날짜는 가능하면 `YYYY-MM-DD` 형식을 사용한다. 불명확한 값은 빈칸 대신 `확인 필요`로 남긴다.

## 1. `공문분류목록.csv`

모든 원본 공문을 한 행씩 기록한다.

```csv
원본파일,공문제목,요청기관,분류,관련도,분류근거,마감일,제출물,제출방식,관련업무,원문근거,읽기상태
```

## 2. `내업무체크리스트.csv`

`직접처리`, `공동확인`, `판단보류` 중심으로 작성한다. `참고`는 실행 의무가 있을 때만 포함한다.

```csv
상태,업무명,분류,마감일,먼저할일,제출물,제출방식,확인할사람,위험요소,원본파일,원문근거
```

상태 예시:

- `오늘 확인`
- `이번 주 처리`
- `대기`
- `확인 필요`
- `참고`

## 3. `마감캘린더.csv`

마감일이 확실한 항목만 넣는다. 자동 등록용이 아니라 교사가 직접 가져오기 위한 후보 파일이다.

```csv
Subject,Start Date,Start Time,End Date,End Time,Description
```

규칙:

- 마감일이 `확인 필요`인 항목은 넣지 않는다.
- 시간 정보가 없으면 `09:00`부터 `09:30`까지 후보로 둔다.
- 설명에는 담당 역할과 제출 전 확인 사항을 적는다.

## 4. `이번주_업무보고서.md`

```md
# 이번 주 공문 업무 보고서

## 한 줄 요약

## 오늘 먼저 볼 공문

## 이번 주 마감 업무

## 직접 처리할 공문

## 공동 확인이 필요한 공문

## 참고하면 좋은 공문

## 판단 보류 공문

## 누락 위험과 확인 필요

## 생성 파일
```

## 5. `확인질문목록.md`

교사가 원문, 업무포털, 관리자, 담당자에게 확인해야 할 질문만 모은다.

```md
# 확인 질문 목록

## 담당 여부 확인

## 마감 확인

## 제출 방식 확인

## 첨부파일 확인

## 개인정보 확인
```

## 6. `공문별_작업안내.md`

각 관련 공문에 대해 비서처럼 행동 안내를 제공한다.

```md
# 공문별 작업 안내

## 공문 제목

- 분류:
- 왜 나와 관련 있는가:
- 마감:
- 제출물:
- 제출 방식:
- 먼저 할 일:
- 확인할 사람:
- 위험 요소:
- 원문 근거:
```

## 7. 검증 요약

작업 마지막에 `이번주_업무보고서.md` 하단 또는 별도 섹션에 다음 값을 남긴다.

- 원본 공문 수
- `공문분류목록.csv` 행 수
- `selected/`에 복사한 파일 수
- 마감일 확인 필요 항목 수
- 읽기 제한 파일 수
- 자동 제출, 자동 발송, 업무포털 입력을 하지 않았는지 여부
