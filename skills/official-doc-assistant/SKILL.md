# Official Document Assistant

이 Skill은 교사가 `raw/` 폴더에 모은 공문을 `data/내업무프로필.md` 기준으로 분류하고, 관련 공문을 `selected/`에 복사한 뒤, `outputs/`에 업무 보고서와 체크리스트를 만든다.

## 기본 원칙

- 한국어로 작업하고 산출물을 작성한다.
- `raw/` 원본 공문은 이동, 삭제, 수정, 이름 변경하지 않는다.
- 관련 공문은 `selected/` 아래 범주 폴더에 복사본으로 분류한다.
- 공문 제출, 자동 발송, 업무포털 입력, 캘린더 자동 등록은 하지 않는다.
- 판단이 애매하면 억지로 확정하지 말고 `판단보류` 또는 `확인 필요`로 남긴다.
- 모든 분류와 주요 추출 항목에는 원본 파일명, 문단, 페이지, 표제 등 확인 가능한 근거를 남긴다.
- 실제 학생, 학부모, 교직원 개인정보가 있으면 공개 산출물에 그대로 쓰지 말고 익명화하거나 `개인정보 포함 - 원문 확인 필요`로 표시한다.

## 프로젝트 구조

작업 폴더는 다음 구조를 기본으로 한다. 없으면 사용자에게 만들도록 안내하거나 안전하게 생성한다.

```text
project/
  data/
    내업무프로필.md
  raw/
  selected/
    직접처리/
    공동확인/
    참고/
    판단보류/
  outputs/
```

템플릿이 필요하면 `assets/project-template/`를 복사해 새 작업 폴더로 사용한다.

## 참고 파일

- 분류 기준이 필요하면 `references/공문분류-기준.md`를 읽는다.
- 출력 파일의 열과 문서 구조가 필요하면 `references/산출물-형식.md`를 읽는다.

## 작업 순서

1. 작업 폴더의 루트를 확인한다. `data/`, `raw/`, `selected/`, `outputs/`가 같은 루트 아래 있어야 한다.
2. `data/내업무프로필.md`를 읽고 담당 학년, 교과, 업무분장, 담당 행사, 관심사, 제외 범위, 확인 관계를 요약한다.
3. `raw/`의 파일 목록을 만든다. 파일 수, 확장자, 읽기 가능 여부를 기록한다.
4. 각 공문에서 제목, 요청 기관, 핵심 요청, 마감일, 제출물, 제출 방식, 관련 업무, 원문 근거를 추출한다.
5. 각 공문을 다음 범주 중 하나로 분류한다.
   - `직접처리`: 내 업무 프로필과 명확히 연결되고 내가 처리할 가능성이 높음
   - `공동확인`: 내 업무와 관련 있지만 다른 담당자나 관리자 확인이 필요함
   - `참고`: 제출 의무는 낮지만 관심사, 수업, 연수, 정책 이해에 유용함
   - `전달필요`: 다른 담당자가 처리할 가능성이 높아 전달 또는 공유가 필요함
   - `무관`: 내 업무, 학년, 교과, 관심사와 관련이 낮음
   - `판단보류`: 공문 내용만으로 담당 여부를 확정하기 어려움
6. `직접처리`, `공동확인`, `참고`, `판단보류` 파일은 `selected/`의 해당 폴더에 복사한다. `전달필요`와 `무관`은 기본적으로 복사하지 않되, 사용자가 원하면 폴더를 추가한다.
7. `outputs/공문분류목록.csv`를 만든다. 모든 공문을 포함한다.
8. `outputs/내업무체크리스트.csv`를 만든다. `직접처리`, `공동확인`, `판단보류` 중심으로 작성한다.
9. 마감일이 확실한 항목만 `outputs/마감캘린더.csv`에 넣는다.
10. `outputs/이번주_업무보고서.md`, `outputs/확인질문목록.md`, `outputs/공문별_작업안내.md`를 작성한다.
11. 마지막에 원본 공문 수, 분류 행 수, 복사된 파일 수, 확인 필요 항목 수를 대조해 검증 결과를 남긴다.

## 파일 복사 규칙

- 복사 전 대상 경로가 작업 폴더 내부인지 확인한다.
- 원본 파일명은 유지한다. 같은 이름이 충돌하면 `_2`, `_3` 같은 접미사를 붙인다.
- 파일 내용을 편집하지 않는다.
- 읽기 어려운 PDF, HWPX, 이미지 공문은 `읽기 제한`으로 표시하고 교사 확인 질문에 올린다.

## 보고서 톤

보고서는 비서처럼 간결하게 쓴다. 공문 내용을 길게 요약하기보다 교사가 바로 행동할 수 있게 다음을 우선한다.

- 오늘 먼저 확인할 일
- 이번 주 마감 업무
- 제출 전 필요한 확인
- 담당 여부가 애매한 공문
- 누락되었거나 원문 확인이 필요한 첨부

## 완료 조건

완료 시 다음을 사용자에게 보고한다.

- 총 공문 수와 분류 결과 요약
- `selected/`에 복사한 파일 수
- 생성한 `outputs/` 파일 목록
- 마감이 확실한 업무 수와 확인 필요 업무 수
- 자동 제출이나 외부 전송을 하지 않았다는 점
