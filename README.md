# official-doc-assistant-skill

교사용 개인 맞춤 공문 분류 및 업무 안내 Codex Skill입니다.

교사가 공문 파일을 한 폴더에 넣으면 Codex가 `내업무프로필.md`를 기준으로 공문을 분류하고, 관련 공문을 별도 폴더에 모은 뒤 마감기한, 제출물, 해야 할 일, 확인할 사람, 위험 요소를 보고서로 정리하도록 안내합니다.

## 가장 쉬운 사용법: Codex에 저장소 링크 붙여넣기

사용자는 자기 Codex에 아래 문장을 그대로 붙여넣으면 됩니다.

```text
다음 GitHub 저장소의 official-doc-assistant Skill을 설치하고 사용 준비를 도와줘.

https://github.com/Bayesian94/official-doc-assistant-skill

설치한 뒤 project-template을 복사해 새 작업 폴더를 만들고,
data/내업무프로필.md를 내가 수정할 수 있게 안내해줘.
그 다음 raw/ 폴더에 공문을 넣으면 내 업무 프로필 기준으로 공문을 분류하고,
selected/에는 관련 공문 복사본을 모으고,
outputs/에는 공문분류목록, 내업무체크리스트, 마감캘린더,
이번주 업무보고서, 확인질문목록, 공문별 작업안내를 만들어줘.
```

Codex가 설치 위치를 물어보면 보통 아래 위치를 사용하면 됩니다.

- Windows: `%USERPROFILE%\.codex\skills\official-doc-assistant`
- macOS/Linux: `~/.codex/skills/official-doc-assistant`

사용 중인 Codex 환경이 GitHub 링크로 Skill을 바로 설치하지 못하면, 아래의 직접 설치 방법을 따르면 됩니다.

## 이 Skill이 하는 일

이 Skill은 공문을 단순 요약하지 않습니다. 사용자의 업무분장, 학년, 교과, 관심사, 제외할 업무를 기준으로 공문을 나누고, 실제로 무엇을 해야 하는지 안내합니다.

분류 범주는 다음과 같습니다.

- `직접처리`: 내가 처리해야 할 가능성이 높은 공문
- `공동확인`: 나와 관련 있지만 다른 담당자나 관리자 확인이 필요한 공문
- `참고`: 제출 의무는 낮지만 관심사나 수업에 도움이 되는 공문
- `전달필요`: 다른 담당자에게 전달해야 할 가능성이 높은 공문
- `무관`: 내 업무와 관련이 낮은 공문
- `판단보류`: 담당 여부를 확정하기 어려운 공문

## 직접 설치하는 방법

GitHub 링크 설치가 잘 되지 않으면 수동으로 설치할 수 있습니다.

1. 이 저장소를 다운로드하거나 clone합니다.
2. `skills/official-doc-assistant/` 폴더를 Codex Skills 폴더에 복사합니다.
3. Codex를 새로 시작하거나 Skill 목록을 다시 불러옵니다.

## 새 공문 작업 시작하기

1. `skills/official-doc-assistant/assets/project-template/` 폴더를 원하는 위치에 복사합니다.
2. 복사한 폴더의 `data/내업무프로필.md`를 작성합니다.
3. `raw/` 폴더에 공문 파일을 넣습니다.
4. Codex에게 다음처럼 요청합니다.

```text
official-doc-assistant Skill을 사용해 주세요.

raw/ 폴더의 공문을 data/내업무프로필.md 기준으로 분류하고,
관련 공문은 selected/에 복사한 뒤,
outputs/에 업무 보고서, 체크리스트, 마감 목록, 확인 질문 목록을 만들어 주세요.
```

## 내 업무 프로필 작성 예시

`data/내업무프로필.md`에는 아래 정보를 적습니다.

실제 사람 이름을 꼭 넣을 필요는 없습니다. 가능하면 `정보부장`, `교감`, `학년부장`처럼 역할명으로 적는 편이 안전합니다.

```md
# 내 업무 프로필

## 기본 정보

- 담당 학년: 5학년
- 담당 교과: 과학, 실과
- 담임 여부: 담임

## 업무분장

- 주 업무: 정보, 디지털교육, 개인정보보호
- 보조 업무: AI교육 주간 운영
- 담당 행사: 정보통신윤리교육, 디지털 시민성 캠페인
- 관련 위원회: 정보화기기선정위원회

## 관심사

- 관심 연수: AI교육, 디지털 리터러시, 에듀테크
- 관심 교육 분야: 생성형 AI 활용 수업, 개인정보보호 교육
- 수업에 활용하고 싶은 주제: 과학 탐구 데이터 정리, 실과 코딩 활동

## 제외 범위

- 내가 직접 처리하지 않는 업무: 급식, 체육대회, 방과후학교, 보건
- 참고만 하면 되는 업무: 일반 교원 연수 안내, 교육 정책 홍보

## 확인 관계

- 최종 확인자: 교감
- 함께 확인할 사람: 정보부장, 학년부장
```

## 생성되는 결과물

- `outputs/공문분류목록.csv`: 모든 공문의 분류 결과
- `outputs/내업무체크리스트.csv`: 내가 확인하거나 처리해야 할 업무 목록
- `outputs/마감캘린더.csv`: 마감일이 확실한 항목만 모은 캘린더 가져오기 후보
- `outputs/이번주_업무보고서.md`: 비서가 정리한 이번 주 업무 보고서
- `outputs/확인질문목록.md`: 담당 여부, 마감, 제출 방식 등 확인해야 할 질문
- `outputs/공문별_작업안내.md`: 공문별로 먼저 할 일과 위험 요소를 정리한 안내

## 중요한 안전 규칙

- 실제 공문, 학생·학부모·교직원 개인정보, 학교 내부 자료를 이 공개 저장소에 올리지 마세요.
- `raw/` 원본 공문은 이동, 삭제, 수정하지 않습니다.
- 관련 공문은 `selected/`에 복사본으로만 모읍니다.
- Skill은 공문을 제출하거나 발송하거나 업무포털에 입력하지 않습니다.
- 마감일이나 담당 여부가 애매하면 `확인 필요` 또는 `판단보류`로 남깁니다.

## 더미 예시

`examples/dummy-project/`에는 사용 흐름을 보여주기 위한 더미 업무 프로필과 더미 공문이 들어 있습니다. 실제 학교명, 실제 공문, 실제 개인정보를 포함하지 않습니다.
